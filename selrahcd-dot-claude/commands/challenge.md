# Challenge - Paranoid Plan Review

Challenge plans, designs, or artifacts with rigorous review.

## Usage
```
/challenge <artifact>
```

Examples:
```
/challenge plan.md
/challenge "the current implementation approach"
/challenge architecture.md
```

## Process

Invoke the `challenge` skill to perform a paranoid review of the specified artifact, looking for:
- Potential issues and edge cases
- Missing considerations
- Assumptions that may not hold
- Alternative approaches
