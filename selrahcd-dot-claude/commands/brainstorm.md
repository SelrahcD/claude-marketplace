# Brainstorm - Explore Requirements Before Implementation

Explore user intent, requirements, and design before any creative work.

## Usage
```
/brainstorm <topic>
```

Examples:
```
/brainstorm "user authentication flow"
/brainstorm "new dashboard component"
/brainstorm "API rate limiting strategy"
```

## Process

Invoke the `brainstorming` skill to explore:
- User intent and goals
- Requirements and constraints
- Design options and trade-offs
- Implementation considerations

Use this before creating features, building components, adding functionality, or modifying behavior.
